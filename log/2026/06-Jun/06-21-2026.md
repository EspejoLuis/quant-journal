## 🔑 Key Points

- **`#pragma once` only in `.hpp`, never in `.cpp`:** `#pragma once` prevents a header from being included more than once in the same translation unit. `.cpp` files are never `#include`d — each is compiled directly as its own translation unit, so there is nothing to guard against.
- **`const` members must be initialized in the member initializer list:** the constructor body runs after all members are already initialized. A `const` member can only be set in the initializer list — the body cannot re-initialize it. Non-const members can be assigned in the body if needed.
- **Free function over class method for stateless operations (C.4):** C++ Core Guidelines rule C.4 — make a function a member only if it needs access to class internals. `simulateGbmPath` reads `ModelParameters` and `SimulationParameters` as parameters and returns a path; it owns no state. As a free function in `common/pathGeneration.hpp` it is reusable by `McEngine`, `ChooserEngine`, and any future engine — no duplication.
- **QuantLib uses objects (`StochasticProcess` + `PathGenerator<RNG>`) for path generation:** a full strategy pattern where the process is injected at construction and `evolve(t, x, dt, dw)` is called per step. Appropriate for a production library needing runtime polymorphism across models. Our learning engine uses free functions — simpler, sufficient, and directly aligned with CP.1 (stateless path generators).
- **Never return `const&` to a local variable:** `simulateGbmPath` creates `simulatedPrices` on its stack. Returning a `const&` to it would dangle — the local is destroyed when the function returns. The function must return by value; the caller can store `const auto result = simulateGbmPath(...)` to signal non-modification.
- **Antithetic paths bug in `ChooserEngine`:** if `VarianceReduction::Antithetic` is set, `simulateGbmPath` returns `2 × nPaths` paths but the loop runs only `simParams_.nPaths` times — the antithetic paths are silently ignored. Fix: loop over `simulatedPaths.size()` and divide by that, matching `McEngine::price()`.
