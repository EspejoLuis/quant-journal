## 🔑 Key Points

- **Object state vs local variable:** if a value is computed from constructor inputs and fixed for the object's lifetime, it belongs in the object as a member. If it is a scratch buffer local to one algorithm call, it stays local. Grid and FD coefficients are fixed at construction → members. `valuesCurrent` / `valuesNext` change on every `price()` call → locals.
- **Thread safety from read-only state:** moving grid and coefficient computation into the constructor makes `price()` effectively read-only. No mutation of shared state during pricing → multiple threads can call `price()` in parallel with no locks needed. This only works because path generators are already stateless (CP.1).
- **`const` members in a struct delete `operator=`:** declaring struct fields `const` makes the struct non-assignable — `operator=` is implicitly deleted. Prefer `const` on the variable holding the struct (`const PdeParameters pdeParams_`), not on the fields inside it (Con.1).
- **`ptrdiff_t` vs `size_t` for indices:** `size_t` is unsigned — if `std::distance` returns 0 and you compute `idx - 1`, you get `2^64 - 1`, not `-1`. `ptrdiff_t` is signed, so `idx - 1` evaluates to `-1` correctly. Always use `ptrdiff_t` when an index might need to go negative or be compared after subtraction.
- **`std::fill` instead of a loop for constant-value vectors:** for log-grid FD coefficients, $a_i$, $b_i$, $c_i$ are the same at every node. `std::fill(v.begin(), v.end(), val)` replaces an explicit loop — cleaner and communicates intent directly.
- **`std::move` for local → member transfer:** after building `a`, `b`, `c` as local vectors, `pdeCoeffs_.a = std::move(a)` swaps internal pointers in O(1) rather than copying O(N) elements. The local is left in a valid but unspecified state.
- **`lower_bound` + `distance` pattern for closest-index lookup:** `std::lower_bound(v.begin(), v.end(), target)` returns an iterator to the first element $\geq$ target on a sorted vector (binary search, O(log N)). `std::distance(v.begin(), it)` converts the iterator to a signed offset. Together they give the right-bracket index for linear interpolation.
- **CMake `CMAKE_BUILD_TYPE` empty → no debug symbols:** without `-DCMAKE_BUILD_TYPE=Debug`, CMake adds no `-g` flag. The binary contains no DWARF info → LLDB registers breakpoints as pending (locations = 0) and never resolves them. `settings.json cmake.buildType` only applies to the CMake Tools extension — not to `cmake ..` in the terminal.
