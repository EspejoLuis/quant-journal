## рҹ”‘ Key Points

- **CMake `PUBLIC` / `PRIVATE` / `INTERFACE`:** `PUBLIC` on `target_include_directories` propagates the include path to every target that links the library вҖ” so test executables inherit `src/engine`, `src/product`, `src/common` automatically without listing them again. `PRIVATE` keeps it local to the target only; `INTERFACE` propagates to dependents but not the target itself (header-only libraries).
- **`inline` for header-defined functions (ODR):** without `inline`, every `.cpp` that includes the header gets its own compiled definition of the function; the linker sees duplicate symbols and errors. `inline` is a linker hint: "treat all copies as the same definition, keep one." It does NOT mean the compiler will inline the call вҖ” that's a separate optimization decision.
- **`Catch::Approx::epsilon` is a relative fraction:** `epsilon(0.005)` = 0.5% relative tolerance, not a percentage. `margin(m)` is the absolute alternative. Use `epsilon` for price comparisons вҖ” it scales with magnitude; `margin` does not.
- **`pow(a, b)` = `exp(b * log(a))` вҖ” two transcendentals:** precomputing `volSqrtDt = vol * sqrt(dt)` once and using `exp(volSqrtDt * z)` inside the loop cuts each step to one `exp`. Refactoring to `pow(exp(volSqrtDt), z)` looks cleaner but is ~2Г— slower.
- **Antithetic variates:** for every draw Z, price a second path with -Z. Payoffs on paired paths are negatively correlated (normal distribution is symmetric about 0), so their average converges faster than either alone. Same number of RNG draws, half the variance for the same path count.
- **`std::accumulate` with a custom binary op:** the 4th argument is `(acc, x) -> new_acc`. The initial value type sets the accumulation type вҖ” use `0.0` not `0` to avoid silent integer truncation of doubles.
- **Hoisting branches out of hot loops:** a flag like `isAntithetic` that never changes inside a loop should be checked once outside it. The cleanest approach: two separate loop bodies selected by an `if/else` before the loops, not `if` inside.
