## рҹ”‘ Key Points

- **Per-thread RNG (CP.20):** `std::mt19937` has a 624-integer mutable state вҖ” two threads calling it simultaneously is a data race. Each thread gets its own seeded instance; paths stay independent because different seeds produce uncorrelated sequences.
- **Value vs reference members:** storing `const Foo&` as a member creates a dangling reference if the source object is destroyed after construction. Store by value so the class owns its data.
- **`const` method placement:** `const` after the parameter list marks the method as non-mutating (e.g. `double payoff(...) const`); `const` before the return type qualifies the return value вҖ” a different (usually useless) thing.
- **Validation in constructor (fail fast):** QuantLib validates in the constructor via `QL_REQUIRE`; invalid objects are never created. Moves the error to the earliest possible point and keeps `price()` clean.
- **Per-path discount factor:** computing `exp(-rT)` inside the loop (rather than once outside) costs nothing with flat rates but makes swapping in a stochastic discount factor trivial later.
- **Jensen's inequality in MC tests:** `E[max(S_T - K, 0)] вү  max(E[S_T] - K, 0)` вҖ” you cannot move an expectation inside a nonlinear function. Using `E[S_T] - K` as the benchmark is only approximately right for deep-ITM options.
