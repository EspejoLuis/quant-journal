## 🔑 Key Points

- **Nested MC at decision points — why hybrid is better:** a product with an intermediate decision date (chooser at $T_c$, cliquet at reset dates) requires knowing the continuation value at that date. Pure MC cannot compute it without launching $N_{\text{inner}}$ sub-paths from each outer path — cost $N_{\text{outer}} \times N_{\text{inner}}$. If the continuation value has a closed-form (BS here), use it: outer MC (0 → $T_c$) + inner BS ($T_c$ → $T$ analytic) gives the exact conditional expectation with no extra variance. This principle generalises: whenever a decision point has an analytic continuation value, hybrid beats nested MC.
- **`CHECK` vs `REQUIRE` in Catch2:** both report failures. `REQUIRE` aborts the test immediately; `CHECK` continues running all remaining assertions. Use `CHECK` when you want to collect all failures in one run (e.g. testing three fields of a getter), use `REQUIRE` when subsequent assertions are meaningless if the first fails.
- **Coverage forces the `parameters()` getter test for non-`payoff()` products:** for `VanillaEuropeanOption` and `DigitalEuropeanOption`, `payoff()` tests implicitly verify the constructor stored values (the payoff uses them). For `ChooserEuropeanOption`, `payoff()` throws without reading stored data — nothing else exercises the getter or the happy path of `validateInputs()`. The `parameters()` test is not optional, it is the only path to 100% coverage.
- **Chooser closed-form scaling factor — $e^{-q(T-T_c)}$ not $e^{-qT_c}$:** after applying put-call parity, $\max(0, P-C) = e^{-q(T-T_c)} \max(0, K e^{-(r-q)(T-T_c)} - S_{T_c})$. The factor $e^{-q(T-T_c)}$ comes from factoring the dividend term out of the put payoff — it is the remaining life $T - T_c$, not the choice date $T_c$. With $q=0$ both expressions equal 1 and the bug is invisible.
